> 담당: songyoungshin-ecommerce(주) + brand-marketing(카피) + sales-distribution(오퍼/구독).

## 0. 프로젝트 목적
단순 쇼핑몰이 아니라 **송영신목장 A2 Jersey Hay Milk 구독 비즈니스를 위한 완전한
온라인 판매 시스템(Online Funnel System)** 구축. shop.a2jerseymilk.com을
**"AI 기반 온라인 깔때기(Funnel) 마케팅 시스템"** 으로 만든다.

## 1. 전체 구조 (역할 분리)
- **브랜드 홈페이지(유지·수정 금지):** www.a2jerseymilk.com (Wix). 역할 = "왜 송영신목장인가?" — 브랜드/A2/Jersey/Hay Milk/목장 스토리/신뢰/콘텐츠.
- **쇼핑몰(개발 대상):** shop.a2jerseymilk.com (GitHub: hhj3150/shop). 역할 = "무엇을 구매할 것인가?" — 판매·정기구독·회원·고객응대·추천·전환.

## 2. 고객 흐름
유입 → Wix 홈페이지 → "쇼핑하기" 클릭 → shop → **고객 유형 분석 → 상품 추천 → 첫 구매 → 정기구독 → 재구매 → 충성 고객**.

## 3. 절대 규칙 (위반 금지)
- Wix 홈/브랜드 홈 구조 **수정 금지**. 수정 대상은 hhj3150/shop 뿐.
- **상품은 2카테고리·4 SKU만.** 카이막/아이스크림/밀크티/퇴비 등 추가 금지.
- 가격·영양성분 **지어내지 말 것** → `lib/products.ts` 실제 데이터 사용. 구독 금액도 데이터로 계산.
- A2 표현은 "우유가 불편했던 분도 비교적 편하게" 수준까지만. **질병치료·의학적 효능 주장 금지.**
- 프리미엄 목장 톤 유지, 기존 디자인 토큰/컴포넌트·**기존 작업 보존**.
- **한 번에 다 고치지 말 것.** 각 단계 완료 후 보고하고 다음 단계로(아래 §9 개발 순서).

## 4. 판매 상품 (4 SKU)
- **milk-180** A2 저지 헤이밀크 180mL — 첫 경험·아이용·맛보기
- **milk-750** A2 저지 헤이밀크 750mL — 메인·가족용·**구독 핵심**
- **yogurt-180** A2 저지 플레인 요거트 180mL — 아침 루틴·아이 간식
- **yogurt-500** A2 저지 플레인 요거트 500mL — 가족용·**세트 구독 핵심**

## 5. 정기구독 구조 (4주 고정 → 3단 플랜)
| 플랜 | 기간 | 배송 | 할인 | 포지션 |
|---|---|---|---|---|
| **Starter** | 4주 | 4회 | **10%** | 체험형 |
| **Premium** | 8주 | 8회 | **15%** | **기본 추천(여기로 유도)** |
| **Founder** | 12주 | 12회 | **20%** | 최고 혜택 |
- 설계 목표: **8주 Premium이 가장 많이 선택되도록** UI/기본값/추천을 설계(앵커링·기본선택·뱃지).
- 운영 규칙(유지): 주 1회 배송 · 회당 배송비 4,000원 · 무통장입금 확인 후 시작 · (한정수량 정책이 있으면 표기).

## 6. 쇼핑몰 퍼널 구조 (페이지/섹션)
1) Hero 2) Funnel Graphic 3) 고객 유형 진단 4) 추천 상품 5) 상품 상세 6) 정기구독 선택 7) 장바구니 8) 신청 9) 재구매.
- Hero 제목: "우유가 불편했던 분들을 위한 국내산 A2 저지 헤이밀크"
- Hero 서브: "송영신목장의 A2 저지 우유와 플레인 요거트를 매주 신선하게 받아보세요."
- Hero CTA: 첫 구매하기 / 정기구독 시작하기 / 내게 맞는 제품 추천받기

## 7. 고객 추천 시스템 (Quiz) — "어떤 이유로 찾아오셨나요?"
1. 우유 마시면 속이 불편해요 → **milk-180 + milk-750(Premium)**
2. 아이에게 좋은 우유 → **milk-180 + yogurt-180**
3. 가족이 함께 마실 우유 → **milk-750(Premium)**
4. 아침 건강 루틴 → **yogurt-180 + yogurt-500**
5. 정기배송을 원해요 → **milk-750 + yogurt-500 + 12주 Founder**
- 선택 결과 → 추천 섹션 동적 노출 + 해당 CTA/플랜 강조 + 선택값 이벤트/ref 기록.

## 8. AI 고객응대 — "A2 Milk Concierge"
- 역할: 상품/구독/배송 안내, FAQ, 상품·구독 추천, 고객 문의 수집.
- **금지: 질병 치료·의학적 효능·법률 자문·환불 분쟁 판단.** 필요 시 **관리자에게 전달(에스컬레이션)**.
- MVP로 구현(룰/스크립트 기반 가능). 응답은 보유 데이터·FAQ 범위 내.

## 9. 상품 카드 / 구독밴드 표기
- 상품 카드 필수: 제품명·용량·정가·회원가·구독 예상금액(데이터 계산)·추천 대상(역할 배지)·"첫 구매" CTA·"정기구독" CTA.
- ProductShowcase 헤드라인: "우유 2종, 요거트 2종. 매주 반복되는 건강한 루틴."
- 구독밴드: 3단 플랜 비교(할인·배송횟수) + Premium 기본 강조 + 운영규칙(주1회·배송비4,000원·무통장입금).

## 10. 유입 추적 (ref) — 필수
- 채널 예: `?ref=wix` `?ref=instagram` `?ref=blog` `?ref=youtube` `?ref=qr` `?ref=cafe`.
- 최초 진입 시 localStorage 저장(기존값 유지정책 명시) → 모든 CTA 클릭·주문·AI상담 payload에 ref 동행. 유틸로 캡슐화.
- 목표: 채널별 유입 → 상품 → 구독 플랜 선택까지 분석 가능.

## 11. 관리자 통계 (확인 가능해야 함)
4주/8주/12주 구독 수 · 상품별 신청 수 · 요일별 신청 수 · ref별 유입 수 · ref별 전환 수 · AI 상담 수 · AI 상담 후 구매 전환 수.

## 12. 추가 구현(전환)
- 모바일 하단 sticky CTA(정기구독/첫 구매 고정).
- SEO 메타(쇼핑몰/구독 중심) + 상품 JSON-LD(Product/Offer).
- 접근성(라벨·대비·키보드), 성능(이미지 최적화·CLS 방지).
- 디자인: 프리미엄 목장 톤 + "데이터 기반 추천·구독 전환" 느낌 일부. 과한 사이버/네온 금지.
