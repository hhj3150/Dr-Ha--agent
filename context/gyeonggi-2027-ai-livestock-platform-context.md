## 0. 한눈 요약
경기도가 보유·접근 가능한 공공·민간·ICT 센서 데이터를 통합·표준화하여, 사람은 쉽게 쓰고 AI는
즉시 학습·분석하는 **「경기도 인공지능 축산행정 플랫폼」**. 지향: 사후 대응 → **예측·예방·정밀 행정(AX)** 전환.
낙농 ICT에서 출발해 한우·양돈으로 확산하는 **5개년** 로드맵.

| 구분 | 핵심 |
|---|---|
| 비전 | 축산 데이터 통합 플랫폼을 통한 경기 축산행정의 AI 대전환(AX) |
| 방식 | 데이터 통합·표준화 → AI 분석·시각화 → 예측·예방·정밀 행정 |
| 출발 | 낙농 ICT 건강센서 파일럿(2~3개 시군·750두) → 본 확산(31개 시군·6,750두) |
| 예산 | **V1.1.0 기준 5개년 총 189.2억원** (※ 위 충돌 주석 참조) |

## 1. 추진 배경·목적
- 경기도 축산 위상(2025.8 이력제 기준): 전국 대비 젖소 사육두수 **40.3%(1위)**, 낙농 농가 **38.7%**,
  육우 농가 **37.4%**, 한우 농가 **8.3%**. → 경기 축산정책 혁신이 전국 표준 모델.
- 문제: ① 데이터 단절(ICT로 데이터는 쌓이나 행정과 실시간 연계 부재) ② 사후 행정(사후 보고에 머물러
  신속성·과학적 근거 한계).
- 목적: AI 축산행정 대전환(AX) — 데이터 기반 의사결정(수립–실행–평가) 구조 확립 + 통합 아키텍처 설계 +
  단/중/장기 로드맵 및 5개년 예산.
- 축산정책 특수성: ① 생물 대상·수년 소요 → 정책 판단 시차 큼 ② 사육두수·사료비·질병·수입량 변동 多 →

## 4. 핵심 ICT 생체센서 데이터 명세 (SmaXtec)
### 하드웨어·행정 효용
- 방식: 벌집위(Reticulum) **1회 투여 평생 유지 볼러스(Bolus)** → 탈락·분실 위험 0, 유지관리 행정비 제로.
- 효용: 법정 전염병 외 **유방염·케토시스·유열·제엽염 등 '소모성 질병'의 선제적 사전예방 행정** 근거 마련.
### 수집 주기·한계
- 목장 목록/기본정보 24h/6h · 동물정보 1h · **센서상태/측정/이벤트알람 10분**.
- 제약: 대량 개체(예 10,000두) 실시간 동시호출 시 API 속도저하 → **본사 파이프라인 기술제휴·배치 최적화 필수.**
### 스키마 (주요 필드)
- **Organisation**: sequence_number, organisation_id(_id, MongoDB ObjectID 24자리), name, role(Admin/Manager),
  timezone_continent/region, active_sensors, inactive_sensors, total_animals.
- **Animal**: sequence_number, _id, name/management_number, **trace_number(이력제 12자리)**, ear_tag/sensor_number,
  birth_date, monthsOld, count_heats(누적 발정), count_lactations(누적 분만), count_lifecycle_cycles(착유-건유 사이클).
### 주요 메트릭 임계값
- `act_index`(활동량): 급증 >70% → 발정 판단·인공수정 적기 알림.
- `temp`(체온): 정상 38.0~39.5°C, 초과 또는 <38°C → 질병알람·수의 연계.
- `calving_index`(분만): >85% → 24~48h 내 분만 예측·분만실 준비.
- `ph`(산도): 정상 6.0~7.0, <5.5 → 급성/만성 산증 알림·사료배합 조정.
- `rum_index`(반추): 정상 400~600분/일, <400분 → 소화기 질환 조기감지.
- `water_intake`(음수량): 정상 70~120L/일, <50L → 질병 또는 급수시설 고장.
### 주요 이벤트 코드
- 번식·분만: heat / heat_dnb(수정금지) / insemination / no_insemination / pregnancy_result / calving_detection /
  calving_confirmation / dry_off.
- 환경·대사·건강: health_606~609(그룹 열스트레스, THI≥72/78/82 단계) / health_703(활동량 감소) /
  feeding_202(산증, pH<5.5·프리미엄 전용) / feeding_311~315(생애주기별 반추 미달·SX.2 전용) /
  feeding_316(그룹 반추감소·TMR 불량) / fertility_105(분만 임박, 24–48h).

## 5. 플랫폼 아키텍처·구현
- **수집계층**: OpenAPI·SFTP 배치·파일 업로드 엔진 연계.
- **처리계층**: 표준화 스키마 매핑·메타데이터 정의 + 중복·오류 자동검증 정합성 엔진.
- **저장계층**: 센서 시계열용 데이터 레이크 + DW(PostgreSQL·SQLite 등).
- **시각화/UI**: 상용 컴포넌트(DevExpress·Syncfusion·Telerik)·BI(Tableau·PowerBI) 임베디드 웹표준.
- **AI/생성형**: OpenAI·Claude·DeepSeek 등 LLM.
  - **MCP**: AI 에이전트 ↔ 경기도 축산DB 표준연결. 공무원 자연어 질문 → AI가 DB 조회·시각화·보고서 초안.
  - **RAG**: 할루시네이션 방지. 사료관리법·경기도 축산시책 추진계획 등 내부 법령/지침을 벡터DB화.
- **정책 시뮬레이션(Applied Econometrics)**:
  - **DID(이중차분)**: 덴마크 외국인노동자 세제(Kleven et al., 2013) 원용. ICT 선정 vs 탈락 농가 생산성 비교 → 순수 정책기여도.
  - **RD(회귀불연속)**: 일본 70세 본인부담 조정(Shigeoka, 2014) 원용. 사육두수 규모별 규제·보조금 기준선 전후 행동변화.

## ★ 우리(D2O/하현제)의 2대 핵심 진입축 (전략 기준 — 모든 관련 에이전트 적용)
이 사업에서 우리의 역할은 정확히 두 가지다. 모든 제안·전략·문서는 이 2축 중심으로 설계한다.

1. **국내 SmaXtec 위내센서 공급자** — 하현제/D2O가 한국 내 SmaXtec(스마스텍) 센서 공급 채널이다.
