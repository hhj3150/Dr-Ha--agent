## 0. 이 자리의 목적 (한 줄)
**"경기도 189.2억 AI 축산행정 플랫폼은 개념이 아니라, 송영신목장에서 이미 돌아가는 현실이다 — 현장이 곧 증거다."**

인수위 정책분과에게 각인시킬 단 하나: *"새 도정의 축산 AX(행정 대전환) 공약을, 가장 빠르고 안전하게
실증·확산시킬 준비된 현장과 파트너가 경기도 안에 있다."*

### 발표 3대 메시지 (Take-home 3)
1. **정책 명분** — 경기도는 전국 젖소의 40.3%. 사후대응 행정을 예측·예방으로 바꿀 법적 근거(데이터기반행정법)와 정책연구가 이미 완비.
2. **준비된 현장** — 송영신목장 = 센서(SmaXtec)+AI(CowTalk)+순환농업(D2O)이 한 곳에서 작동하는 **즉시 가동형 실증지(Living Lab)**.
3. **확산 설계** — 7,750두 RCT → 한우·양돈 → 국비 건의 → 타 시도 모델. 일자리·탄소·축산경쟁력까지 연결되는 5개년 로드맵.

## 2. 슬라이드 구성안 (브리핑룸 발표 — 9장)

### 슬라이드 1 — 표지
- 제목: **"경기도 축산행정, 예측의 시대로"**
- 부제: AI 축산행정 플랫폼 실증 현장 — 송영신목장
- 하단: 농업회사법인 ㈜D2O · CowTalk AI · Eco-BIT / 발표 하현제
- 비주얼: 목장 전경 + 대시보드 화면 합성 1컷

### 슬라이드 2 — 왜 경기도인가 (정책 명분)
- 핵심 수치(보고서 근거):
  - 경기도 = 전국 젖소 사육두수 **40.3%**, 낙농 농가 **38.7%** (전국 최대 축산 기반)
  - 유우검정 경기 **1,346농가 · 72,642두** 데이터 보유
- 문제 정의: ICT로 데이터는 쌓이나 **행정과 실시간 연계 부재 → 대부분 사후 보고** → 정책 신속성·근거 부족
- 한 줄: *"가장 큰 축산 데이터를 가진 道가, 가장 늦게 반응하고 있다."*

### 슬라이드 3 — 무엇을 바꾸는가 (Before → After)
| | AS-IS (사후대응) | TO-BE (예측·예방·정밀) |
|---|---|---|
| 질병 | 발병 후 신고·살처분 | 사전징후 탐지·예방 행정 |
| 데이터 | 흩어짐·사후 보고 | 통합·표준화·실시간 |
| 정책 | 경험·감(感) | 근거기반(RCT 실증) |
- 법적 근거: 「데이터기반행정 활성화에 관한 법률」(2020.12 시행)
- 해외 전환 사례: 아일랜드 '처벌→사전 알림·협력'(2025 알림 **81% 기한 내 소명**)

### 슬라이드 4 — 경기도의 청사진 (189.2억 / 5개년)
- 「경기도 AI 축산행정 플랫폼」 정책연구(Antller, 발주 경기도, 최종보고 V1.1.0) 요약
- 구조: 공공·ICT·행정 데이터 → 표준화 → AI 분석·시각화 → **예측·예방·정밀 행정**
- 출발: 낙농 ICT 시범사업 — 파일럿 2~3시군 750두 → 확산 31시군 6,750두(RCT 설계 7,750두) → 한우·양돈
- 예산: **5개년 총 189.2억원** (1차년 집중 102.9억 → 5차년 19.4억)
- 메시지: *"청사진은 이미 있다. 남은 건 '제대로 실증할 현장'이다."*

### 슬라이드 5 — 그 현장이 여기 있습니다 (송영신목장)
- 송영신목장 = 센서·AI·순환농업이 **한 곳에서 동시에 작동하는 유일한 통합 실증지**
- 보유 역량: 제네틱스(수정란·번식 18년)·고려동물병원(수의 4인)·D2O(환경기술)·A2 Hay Milk(6차산업)
- 철학: 건강한 흙 → 건강한 풀 → 건강한 소 → 좋은 우유 → 퇴비 → 흙 (완전 순환)
- 한 줄: *"파일럿을 위해 새로 만들 필요가 없다. 이미 돌아가고 있다."*

### 슬라이드 6 — 어떻게 작동하나 (3-레이어)
- **① 센서(입력) — SmaXtec 반추위 센서**: 반추·활동·체온·pH·음수 → 유방염·케토시스·유열·제엽염 사전징후
- **② AI(분석) — CowTalk AI**: 발정·수정적기·임신·분만·질병위험·이상탐지 예측 → 경보
- **③ 연동(행정) — Eco-BIT**: 표준화·집계·익명화 → 경기도 행정 대시보드(RAG·MCP)
- 핵심 차별점: 정책연구가 명시한 **"SmaXtec 센서 + 본사 기술 제휴 필수"의 국내 주체가 우리** = 대체 불가 채널
- (※ "센서도 우리, 두뇌도 우리" — 단일 책임·낮은 통합 리스크·빠른 실증)

### 슬라이드 7 — 무엇을 증명할 것인가 (RCT 실증 설계)
- 설계: 처치군(센서+CowTalk) vs 대조군(비도입) **무작위비교시험**
- 측정지표(파일럿 목표):
  - 소모성 질병 **조기검출률 ↑ / 발병률 ↓**
  - 발정 발견율·**수태율 ↑**, 분만사고 ↓
  - 행정 인지~대응 **소요시간 ↓**, 농가 노동·비용 ↓
- 의미: *"정책 효과를 '느낌'이 아니라 '데이터'로 증명한다."* (오레곤 메디케이드 RCT 방식)

### 슬라이드 8 — 그래서 경기도가 얻는 것 (정책 성과)
- **축산경쟁력**: 생산성·번식효율↑, 질병 손실↓ → 농가 소득
- **재난·방역**: 실시간 관제, 질병 사전대응 → 살처분·예산 낭비 최소화
- **탄소·ESG**: 순환농업·메탄 관리 → 道 탄소중립 실적
- **일자리·6차산업**: 센서 설치·운영·데이터 인력 + A2 브랜드 가공·관광
- **확산성**: 낙농 → 한우·양돈 → 국비 건의 → **타 시도 표준모델(경기 발(發) 전국화)**

### 슬라이드 9 — 제안 (인수위에 드리는 3가지)
1. 송영신목장을 **도(道) 공식 축산 AX 실증 1호 현장(Living Lab)**으로 지정 검토
2. 낙농 ICT 시범사업(7,750두)을 **새 도정 100일 내 착수 과제** 후보로 검토
3. 인수위–D2O 실무 **후속 미팅** 일정 합의 (기술검증·예산·확산 로드맵 상세 브리핑)
- 마무리 한 줄: *"경기도의 축산 AX, 가장 빠른 길은 이미 검증된 현장에서 시작하는 것입니다."*

## 3. 현장 시연 스테이션 대본 (③ 단계 — 25분)

> 원칙: **각 스테이션 3~5분, "보여주고 → 한 문장으로 정책 의미 연결".** 전문용어는 1개당 1개만, 즉시 풀어 말한다.

### 스테이션 A — 축사 / 센서 부착 소 (5분)
- 보여줄 것: 실제 반추위 센서(볼러스)를 가진 소, 개체-센서 매핑
- 대본: *"이 소 뱃속(첫째 위)에 동전만 한 센서가 들어 있습니다. 사람으로 치면 24시간 건강검진입니다.
  체온·되새김·움직임·물 마시는 양을 1분 단위로 봅니다. 사람이 보기 전에 소가 먼저 '아프다'고 말합니다."*
- 정책 연결: *"이게 바로 '사후대응 → 예측'의 출발점입니다."*

### 스테이션 B — 라이브 대시보드 (CowTalk AI) (7분) ★핵심
- 보여줄 것: 실시간 화면 — 발정·질병위험·분만예측 경보, 6대 질문(무엇이/왜/조치/언제/누구에게/표현)
- 대본: *"화면을 보십시오. 어제 이 소가 '발정 신호'를 보냈고, 시스템이 '수정 적기 D-1'을 알렸습니다.
  수정사에게는 적기를, 수의사에게는 임상 소견을, 농가에는 '오늘 할 일'을 따로 보냅니다."*
- 정책 연결: *"농가가 보는 화면과 '경기도 행정이 보는 화면'은 다릅니다. 행정용은 익명·집계 형태로,
  시군별 질병 위험을 한눈에 봅니다. 이것이 189.2억 플랫폼의 행정 대시보드 원형입니다."*

### 스테이션 C — 데이터가 행정으로 가는 길 (Eco-BIT) (5분)
- 보여줄 것: 센서 → 표준화 → 집계·익명화 → 행정 대시보드 흐름 1장 도식
- 대본: *"농가 데이터를 그대로 정부에 주지 않습니다. 표준화하고 익명·집계해서, 개인정보 걱정 없이
  '지역 단위 위험'만 행정에 전달합니다. RAG로 담당자가 바뀌어도 지식이 끊기지 않고, 자연어로 물으면
  근거까지 붙은 답을 줍니다."*
- 정책 연결: *"데이터 권한·보안 문제를 설계 단계에서 이미 풀었습니다."*

### 스테이션 D — 순환농업·브랜드 (D2O 퇴비장 / A2) (선택, 5분)
- 보여줄 것: D2O 베딩·퇴비장, A2 Hay Milk 제품
- 대본: *"소가 건강해지면 분뇨가 자원이 됩니다. D2O 기술로 냄새를 줄이고 프리미엄 퇴비로 되돌립니다.
  그 우유는 A2 헤이밀크 브랜드가 됩니다. 센서 한 개가 질병만이 아니라 탄소·민원·농가소득까지 바꿉니다."*
- 정책 연결: *"축산 AX는 '디지털'만이 아니라 '탄소·6차산업·일자리'까지 닿습니다."*

## 4. 예상 질문 & 대응 (Q&A 준비)

| 인수위 예상 질문 | 핵심 답변 (요지) |
|---|---|
| "이미 다른 ICT 사업도 많은데 뭐가 다른가?" | 범용 생성형AI가 아니라 **축산 특화 정형데이터** 통합·예측. 센서 입력단과 AI 분석단을 한 주체가 책임 → 통합 리스크 최소. |
| "SmaXtec은 외국 센서인데 종속 우려는?" | 국내 공급·API 채널을 우리가 확보. 데이터는 Eco-BIT에서 국내 표준화·보관. 센서는 수단, **두뇌(CowTalk)는 국산.** |
| "데이터 개인정보·권한 문제는?" | 농가용/행정용 경로 분리, 행정용은 **집계·익명화**. 혈통·유우검정은 종축개량협회·농협 별도 협조 트랙. |
| "예산 189.2억, 효과는 검증되나?" | **RCT(무작위비교시험)**로 처치군 vs 대조군 비교. 수태율·질병·대응시간을 데이터로 증명. (느낌 아님) |
| "농가가 따라올까? (수용성)" | 농가는 '오늘 할 일' 수준의 쉬운 알림만 받음. 송영신목장이 **수용성 실증 레퍼런스.** |
| "다른 시도와 중복 아닌가?" | 경기도가 먼저 표준모델을 만들면 **국비 건의·타 시도 확산의 발신지**가 됨(경기 발 전국화). |
| "비용(농가 부담)은?" | 인프라 목장당 500만원, 센서 두당 월 12,000원(4년·고장 무상교체) — 시범사업은 道 지원 설계. (※ 내부 단가 언급 금지) |
| "당장 시작 가능한가?" | 송영신목장은 **이미 가동 중** → 신규 구축 시간 불필요. 100일 내 착수 가능. |

> ⚠️ 금지선: Dr. Ha Liquid 원리·메탄저감 전략 세부·한우 유전자 수출은 언급 금지. 지분·원가·마진 비공개.
