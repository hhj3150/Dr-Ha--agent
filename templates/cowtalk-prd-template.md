# CowTalk AI — PRD Template

> CowTalk AI feature within the Eco-BIT platform.

## 1. Feature name

## 2. Problem / opportunity
What decision is hard today for which user?

## 3. Target users
Farmer / Veterinarian / Inseminator / Farm manager / Government / Disease-control authority

## 4. Sensor & data inputs
Signals used (rumination, activity, body temp, pH, drinking, feeding, DIM, pregnancy status, parity, calving history, herd/farm trends).

## 5. Detection logic
Rules + thresholds + model approach (with confidence/uncertainty).

## 6. The six answers
1. What is happening 2. Why important 3. Action 4. Timing 5. Who notified 6. Dashboard view

## 7. User journeys
Per role: trigger → notification → action → resolution.

## 8. Alert spec
Category, severity, confidence, delivery channel, dedup/escalation.

## 9. API & data model
Endpoints, payloads, tables/measurements (PostgreSQL / TimescaleDB).

## 10. Dashboard requirements
Cards, trends, drill-downs; farm-facing vs government-facing separation.

## 11. Eco-BIT integration
How this plugs into the Eco-BIT platform shell and dashboards.

## 12. Validation & KPIs
Accuracy/precision/recall targets, false-alert rate, pilot validation plan.

## 13. Risks & confidentiality
Clinical caveats, data privacy, confidentiality rules.

## 14. Implementation tickets
Small Claude Code tasks with acceptance criteria and tests.
