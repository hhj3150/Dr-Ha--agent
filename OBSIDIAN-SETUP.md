# 옵시디언 연결 가이드 (회장님용)

> 이 저장소는 **이미 옵시디언 볼트(Vault)** 입니다. 17개 에이전트·context·템플릿이
> 전부 마크다운이라, 옵시디언으로 열면 그대로 D2O 지식베이스가 됩니다.
> 하실장(Claude)과 옵시디언을 잇는 방법을 추천 순서로 정리했습니다.

---

## 한눈에 — 어떤 구조인가

```
[회장님: 옵시디언에서 보고·메모] ⇄ [GitHub 저장소] ⇄ [하실장(Claude Code): 작성·실행]
```

- 회장님은 **옵시디언에서 보고/검색/메모**만 하시면 됩니다.
- 무거운 작성(제안서·보고서·SOP·전략)은 **하실장에게 지시** → 결과가 옵시디언에 노트로 쌓입니다.
- 둘은 같은 마크다운 파일을 보므로 **양방향 동기화**됩니다.

---

## 방법 ① 저장소를 옵시디언 볼트로 열기 (가장 권장 · 5분)

1. PC에 저장소를 내려받습니다.
   ```bash
   git clone https://github.com/hhj3150/dr-ha--agent.git
   ```
2. 옵시디언 실행 → **"폴더를 볼트로 열기(Open folder as vault)"** → 내려받은 `dr-ha--agent` 폴더 선택.
3. 끝. 왼쪽에 `agents/`, `context/`, `templates/`, `workflows/`, `deliverables/`가 노트로 보입니다.
4. **그래프 뷰(Graph view)** 를 켜면 사업 간 연결이 시각화됩니다.

**하실장과의 연결:** 같은 폴더에서 Claude Code를 실행하면, 하실장이 만든 산출물은
옵시디언에 즉시 노트로 뜨고, 회장님이 옵시디언에 적은 메모는 하실장이 바로 읽습니다.

---

## 방법 ② 휴대폰에서도 보기 — iCloud 동기화 (아이폰, 가장 권장·무료·검증됨)

> ✅ 실제로 검증된 방법입니다(2026-06-28, macOS + 아이폰). Mac과 아이폰이 같은
> Apple 계정이면 **iCloud로 무료·자동 동기화**가 가장 쉽고 안정적입니다.

### A. Mac에서 — 볼트를 iCloud 옵시디언 폴더로 옮기기 (한 번만)
1. **Finder** → `Cmd+Shift+G` → 아래 경로 붙여넣고 엔터:
   ```
   ~/Library/Mobile Documents/iCloud~md~obsidian/Documents
   ```
2. 볼트 폴더(예: `Dr-Ha--agent-…`)를 이 창 안으로 **드래그**해 넣기.
3. ☁️ 표시 사라질 때까지 대기(업로드 완료).

### B. Mac 옵시디언에서 — 옮긴 위치로 다시 열기
- 볼트 이름 클릭 → **보관함 폴더 열기** → `Cmd+Shift+G` → 위 경로 → 그 폴더 선택.
- (이렇게 해도 방법 ③ MCP 연동은 그대로 작동 — MCP는 "지금 열린 볼트"에 붙음.)

### C. 아이폰에서
1. App Store → **Obsidian** 설치.
2. 앱 실행 → **iCloud 볼트 목록**에 그 폴더가 보임 → 받기/열기.
3. 1~2분 뒤 `agents/ context/ deliverables/` 가 아이폰에도 표시 → 완성.

→ 이후 Mac에서 하실장이 쓴 노트가 **아이폰에 자동으로** 뜨고, 아이폰 메모도 Mac으로 내려옵니다.

### 대안
- **안드로이드폰** 또는 파일 이동이 번거로우면 → **옵시디언 공식 Sync**(월 약 $4~8,
  기종 무관·켜기만 하면 자동) 또는 **Obsidian Git**(GitHub 저장소와 동기화, 무료이나 모바일 설정 다소 복잡).

## 방법 ③ 옵시디언 MCP로 하실장이 직접 제어 (고급 옵션)

하실장(Claude 데스크톱)이 옵시디언 볼트를 **직접 읽고 쓰게** 만드는 방식.
방법 ①+②로 충분하므로, 자동화를 더 원하실 때만 진행하세요.

### 준비물
- 옵시디언 (방법 ①로 볼트 열어둔 상태)
- Claude 데스크톱 앱
- Node.js 설치 (https://nodejs.org → LTS 버전)

> ✅ 실제로 검증된 방법입니다(2026-06-28, macOS + 옵시디언 1.12.7 기준).
> 핵심: **별도 MCP 패키지(`obsidian-mcp-server` 등)를 쓰지 말 것.** 최신
> "Local REST API **with MCP**" 플러그인(4.x)에는 **MCP 서버가 내장**돼 있어,
> `mcp-remote`로 플러그인에 직접 붙는 게 정답입니다. 별도 패키지는 충돌만 일으킵니다.

### 1단계 — 옵시디언에 "Local REST API with MCP" 설치
1. 옵시디언 → 설정(⚙️) → **커뮤니티 플러그인** → "제한 모드(Restricted mode)" 끄기.
2. **둘러보기(Browse)** → `Local REST API` 검색 → **"Local REST API with MCP"**(만든이 Adam Coddington) 설치 → 활성화.
3. 플러그인 설정에 들어가 **API Key**(긴 문자열)를 **복사**해 둡니다.
4. 주소 확인: 기본 `https://127.0.0.1:27124` (자체 서명 HTTPS).

### 진단(권장) — 옵시디언 쪽이 살아있는지 30초 확인
- 사파리 주소창에 `https://127.0.0.1:27124/` 입력 → 보안 경고는 "계속 진행".
- `{"status":"OK", ... "Local REST API with MCP" ...}` 가 보이면 옵시디언 쪽 정상.
  (`"authenticated": false`는 정상 — 그냥 키를 안 넣고 들어갔을 뿐입니다.)

### 2단계 — Claude 데스크톱에 등록 (mcp-remote로 플러그인에 직접 연결)
Claude 데스크톱 → Settings → Developer → **Edit Config** →
파일 내용을 **전부 지우고** 아래로 교체. `여기에_키` 자리에 1단계 API Key를
(`Bearer ` 뒤 한 칸 띄우고) 넣습니다.

```json
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://127.0.0.1:27124/mcp",
        "--header",
        "Authorization: Bearer 여기에_키"
      ],
      "env": {
        "NODE_TLS_REJECT_UNAUTHORIZED": "0"
      }
    }
  }
}
```

- `NODE_TLS_REJECT_UNAUTHORIZED: "0"` — 자체 서명 인증서를 허용(로컬이라 안전).
- MCP 주소(`/mcp`)가 안 맞으면, 옵시디언 Local REST API 설정 화면을 아래로
  스크롤하면 **MCP 섹션**에 정확한 주소/예시가 있으니 그 값을 쓰세요.

### 3단계 — 확인
1. Claude 데스크톱을 **완전 종료(Cmd+Q) 후 재시작** (옵시디언은 켜둔 채).
2. 첫 실행 때 `npx`가 `mcp-remote`를 받느라 10~20초 걸릴 수 있음 — 기다리세요.
3. "옵시디언 deliverables 폴더에 '연동 테스트' 노트 만들어줘" → 옵시디언에
   노트가 뜨면 **성공**. 이제 하실장이 볼트를 직접 읽고 씁니다.

### 문제 해결
- 도구는 떴는데 읽기·쓰기 전부 실패 → 십중팔구 **별도 패키지(`obsidian-mcp-server`)를
  쓴 경우**. 위 `mcp-remote` 방식으로 교체하면 해결됨.
- 연결 안 됨 → 옵시디언이 **켜져 있어야** 함(플러그인은 앱이 떠 있을 때만 동작).
- SSL/인증서 오류 → `NODE_TLS_REJECT_UNAUTHORIZED`가 `"0"`인지 확인.
- 그래도 안 되면 → 방법 ①+②만으로도 양방향 동기화는 충분합니다.

## 추천 폴더 운영 (옵시디언 안에서)

이미 있는 구조를 그대로 쓰시면 됩니다.

- `agents/` — 17개 에이전트 정의 (수정 거의 안 함)
- `context/` — 창업자·사업 컨텍스트 (사실 바뀌면 갱신)
- `templates/` · `workflows/` — 양식·절차
- `deliverables/` — **하실장이 만든 결과물이 쌓이는 곳** (회장님이 주로 보실 폴더)

옵시디언 메모는 `deliverables/` 안에 날짜 노트(예: `2026-06-28-회의메모.md`)로 적으시면
하실장이 그 맥락을 읽고 이어서 작업합니다.
