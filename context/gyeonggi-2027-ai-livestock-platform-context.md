## 0. 한눈 요약
경기도가 보유·접근 가능한 공공·민간·ICT 센서 데이터를 통합·표준화하여, 사람은 쉽게 쓰고 AI는
즉시 학습·분석하는 **「경기도 인공지능 축산행정 플랫폼」**. 지향: 사후 대응 → **예측·예방·정밀 행정** 전환.
낙농 ICT 시범사업에서 출발해 한우·양돈으로 확산하는 **5개년(총 102.3억원)** 로드맵.

| 구분 | 핵심 |
|---|---|
| 비전 | 축산 데이터 통합 플랫폼을 통한 경기 축산행정의 AI 대전환(AX) |
| 방식 | 데이터 통합·표준화 → AI 분석·시각화 → 예측·예방·정밀 행정 |
| 출발 | 낙농 ICT 건강센서 시범사업(31개 시군 × 5농장, 총 7,750두) |
| 예산 | **5개년 총 102.3억원** |

## 1. 추진 배경·목적
- 경기도 축산 위상: 전국 대비 젖소 사육두수 **40.3%**, 낙농 농가 38.7%, 한우 농가 8.3% (전국 최대 축산 기반).
- 문제: ICT로 데이터는 쌓이나 행정과 실시간 연계 부재 → 대부분 사후 보고 → 정책 신속성·정확성·근거 부족.
- 목적: AI를 축산행정에 접목, 데이터 기반 의사결정 플랫폼 설계 + 단/중/장기 로드맵.
- 법적 근거: 「데이터기반행정 활성화에 관한 법률」(2020.12 시행).

## 2. 국내외 사례
- 미국: 증거기반정책결정위원회법(2019), 연방 AI 활용 1,700건+ 공개, USDA RMA 데이터마이닝 부정수급 1,106만$ 환수.
- EU: 지역모니터링시스템(AMS) 위성+AI 검증, 아일랜드 '처벌→사전 알림·협력' 전환(2025 알림 81% 기한 내 소명).
- 美 국방부: 상용 AI(Gemini) 기반 'GenAI.mil' — 자체개발 대신 검증된 상용 AI 즉시 도입, 멀티클라우드.
- 인터뷰 3대 키워드: ① 정확한(Accurate) ② 통합된(Integrated) ③ 쓰기 쉬운(Easy to use) 데이터.

## 3. 가용 데이터 (보고서 최대 비중)
공공데이터(축평원·API), KOSIS(통계청), 축산유통 다봄(축평원·파일), 경기도 공공데이터(경기데이터드림),
**ICT 데이터(기업 SmaXtec 등·API·권한/전송속도 이슈)**, 혈통(종축개량협회·비공개API), 가축시장(농협경제지주·파일),
유우검정(농협 젖소개량사업소·**경기 1,346농가·72,642두**), 농업위성(농진청·2026 발사예정), 온실가스(기후에너지환경부).

핵심 진단:
- 경기데이터드림에 농림축산해양 **154건** 즉시 투입 가능(2025.12 기준 95개 기관·누적 3,064종 개방).
- 핵심 통계(다봄 등) API 미제공 → 크롤링/배치 또는 축평원 협업 보완.
- **반추위 센서(SmaXtec)** = 소모성 질병(유방염·케토시스·유열·제엽염) 사전예방 행정 근거. 단, 대량 호출 실시간성 한계 → **본사 기술 제휴 필수.**
- 행안부 인허가 데이터는 경기데이터드림과 연동 완료 → 행안부–축평원–경기도 연계로 '중복 없는 행정통계' 가능.

## 4. 플랫폼 설계
핵심 미션: "사람과 AI 모두 편하게 쓰는 축산 행정 데이터 생태계" — 데이터 표준화('공통의 언어').
- 단계: ①기반구축(데이터 활용·리터러시 교육) ②행정 효율화(자동화·AI Assistant·대시보드) ③의사결정 강화(정책 설계·평가) ④정책 고도화(최적화·문화 정착).
- 구조: 공공/유관/ICT/행정 데이터 → 표준화·메타데이터 → DB 통합 → 플랫폼 → ①시각화·분석(PowerBI·Tableau) ②언어모델(RAG·MCP / OpenAI·Claude·Gemini) ③정책연구.
- RAG = '상시 인수인계'(담당자 교체에도 지식 연속성). MCP = 자연어 질문 → AI가 데이터·도구 호출 → 근거 포함 요약·시각화·보고서 초안.
- 정책효과 실증 방법론: 패널분석(덴마크 세제), RDD(일본 의료비), RCT(美 오레곤 메디케이드), 지리경계(캘리포니아 전력) → 경기도 지원·규제 효과를 결과 데이터로 검증.

## 5. 추진 전략 (시범사업·예산·기대효과)
### 낙농 ICT 시범사업 (핵심 실증)
- 목적: 센서 도입이 아니라 낙농 행정을 사후대응 → 예측·예방으로 전환하는 실증.
- 규모: **31개 시군 × 5개 농장, 농장당 착유우 50두, 총 7,750두.**
- 기간: 3단계 36개월 (①ICT 도입·기반 0~6M ②AI 플랫폼·데이터 통합 7~24M ③정책 실증·확산 25~36M).
- 설계: 선정·비선정 농가 데이터 모두 확보 → **무작위비교시험(RCT)**.
- 확산: 한우(이력제·가축시장·번식), 양돈(질병·분뇨·냄새·민원).

### 중복성·기대효과
- 차별화: 경기도 범용 '생성형 AI플랫폼'과 달리 **축산 특화 정형데이터**(사육·질병·유통·수급·가격·환경·온실가스) 통합·분석·시각화.
- 기대: 예산낭비 최소화, 실시간 관제·재난대응, 정책 시뮬레이터, 낙농→타 축종 확산.
- 활용: 「AI 축산행정 대전환 프로젝트」 5개년 실행계획·국비 건의·시군 확산 가이드라인 기반자료.
