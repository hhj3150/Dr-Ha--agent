# Shop Build Brief — shop.a2jerseymilk.com (자기완결형 인수인계서)

> 목적: 별도의 `shop` 저장소 세션에서 작업하는 Claude Code가 이 문서 **하나만** 보고도
> 우리가 합의한 대로 정확히 작업하게 하는 **이동 가능한 작업 지시서**.
> 이 문서는 dr-ha--agent(본부)에 영구 저장되며, 쇼핑몰 세션에 통째로 붙여넣어 사용한다.
> (shop 세션은 dr-ha--agent를 읽을 수 없으므로, 필요한 모든 맥락을 여기 담는다.)

## 0. 한 줄 목표
www.a2jerseymilk.com(Wix 브랜드 홈)에서 "구매하기"로 유입되는 **shop.a2jerseymilk.com**을
**"유입 → 상품 이해 → 추천 → 첫 구매 → 정기구독 → 재구매"** 깔때기형 구독 쇼핑몰로 재설계.

## 1. 절대 규칙 (위반 금지)
- **수정 금지:** Wix 홈페이지(www.a2jerseymilk.com)·브랜드 홈페이지 구조. **수정 대상은 hhj3150/shop, shop.a2jerseymilk.com 뿐.**
- **상품은 2카테고리(우유/요거트) · 4 SKU만.** 카이막·아이스크림·밀크티·퇴비 등 **추가 금지.**
  - `milk-180` A2 저지 헤이밀크 180mL
  - `milk-750` A2 저지 헤이밀크 750mL
  - `yogurt-180` A2 저지 플레인 요거트 180mL
  - `yogurt-500` A2 저지 플레인 요거트 500mL
- **가격·영양성분 지어내지 말 것.** 반드시 `lib/products.ts` 실제 데이터 사용. 구독 예상금액도 데이터로 계산.
- **표현 규칙:** A2는 "우유가 불편했던 분도 비교적 편하게" 수준까지만. **질병 치료·의학적 효능 주장 금지.**
- 기존 **고급 목장 브랜드 톤 유지.** 디자인 토큰/기존 컴포넌트 재사용.
- **작업 브랜치 + 단계별 작은 커밋.** 1단계(분석) 후 멈추고 보고. 승인 전 대규모 수정 금지.

## 2. 브랜드/전략 배경 (왜 이렇게 하나)
- 송영신목장 = 프리미엄 국내산 **A2 저지 헤이밀크** + 플레인 요거트. 철학 "Soil to Soul"(건강한 흙→행복한 소→좋은 우유).
- shop은 감성 브랜드몰이 아니라 **전환·구독 전용 채널.** 핵심 KPI: 첫 구매 전환율, 정기구독 전환율, 객단가, 재구매율.
- 6대 목표: 오픈 준비 · 첫 1,000명 · 정기구독 · 브랜드 · 객단가↑ · 재구매율↑.

## 3. 1단계 — 현재 코드 분석 (먼저 보고, 수정 X)
다음을 읽고 구조·이탈지점을 보고서로 제출 후 **정지**:
1. `app/page.tsx` 메인 섹션 흐름  2. `lib/products.ts` 데이터 구조(가격/회원가/영양/이미지)
3. ProductShowcase 카드/필터/CTA  4. SubscriptionBand 구조  5. `app/products/[id]` 상세
6. `/signup` 또는 결제/신청 흐름  7. 장바구니/주문서  8. ref 파라미터 추적 유무
9. 모바일 CTA 강도(고정 CTA 유무)  10. 퍼널 이탈지점(근거와 함께)

## 4. 2단계 — 재설계 사양

### 4-1. 메인 퍼널 순서 (`app/page.tsx`)
1) Hero 2) FunnelGraphic 3) RecommendationQuiz(고객 유형 진단) 4) 추천 상품(진단 결과 동적)
5) 우유/요거트 카드(ProductShowcase) 6) SubscriptionBand 7) 신뢰 증거 8) FAQ 9) Final CTA

### 4-2. Hero
- 제목: **"우유가 불편했던 분들을 위한 국내산 A2 저지 헤이밀크"**
- 서브: **"송영신목장의 A2 저지 우유와 플레인 요거트를 매주 신선하게 받아보세요."**
- CTA 3: `첫 구매하기` · `정기구독 시작하기` · `내게 맞는 제품 추천받기`(→Quiz 스크롤)

### 4-3. 상품 역할 (카드·추천에 반영)
- **milk-180**: 첫 경험·아이 간식·휴대용·맛보기 / CTA `180mL로 시작하기`, `아이용 우유로 선택`
- **milk-750**: 메인 구독·가족 식탁·주간 정기배송 핵심 / CTA `750mL 정기구독`, `가족용으로 선택`
- **yogurt-180**: 1인 아침 루틴·아이 간식·우유와 추가구매 / CTA `요거트 180mL 선택`, `아침 루틴으로 시작`
- **yogurt-500**: 가족용·세트 구독 보조·우유+요거트 번들 핵심 / CTA `요거트 500mL 선택`, `우유와 함께 구독`

### 4-4. RecommendationQuiz (신규 컴포넌트)
질문 "어떤 이유로 찾아오셨나요?" → 선택 시 추천 SKU 동적 노출 + 해당 CTA 강조:
1. 우유 마시면 속 불편 → milk-180 맛보기 + milk-750 구독
2. 아이에게 좋은 우유 → milk-180 + yogurt-180
3. 가족이 함께 마실 우유 → milk-750 정기구독
4. 아침 건강 루틴 → yogurt-180 + yogurt-500
5. 매주 신선 정기배송 → milk-750 + yogurt-500 세트 구독
→ 선택 결과는 추천 섹션으로 스무스 스크롤, 선택값을 이벤트/ref에 기록.

### 4-5. ProductShowcase
- 헤드라인: ~~"단 네 가지. 곁에 둘 가치가 있는 것만."~~ → **"우유 2종, 요거트 2종. 매주 반복되는 건강한 루틴."**
- 카드 필수 표기: 제품명·용량·**정가**·**회원가**·**1개월 구독 예상금액(데이터 계산)**·추천 대상(역할 배지)·`첫 구매` CTA·`정기구독` CTA
- `ProductRoleBadge`/`RecommendedFor` UI로 역할 노출.

### 4-6. SubscriptionBand (전환형으로 명확화)
감성 톤 유지 + 아래 **명확 표기**: 1개월 단위 · 4주 배송 · **주 1회 배송** · **회원가 10% 할인** ·
회당 배송비 **4,000원** · **요일별 100명, 총 500명 한정** · **무통장입금 확인 후 시작**.

### 4-7. 추가 구현
- 신규 컴포넌트: `RecommendationQuiz`, `FunnelGraphic`, `ProductRoleBadge`/`RecommendedFor`
- **ref 추적:** URL `?ref=` 최초 진입 시 localStorage 저장(기존값 유지정책 명시), 모든 CTA 클릭·주문 payload에 ref 동행. 유틸로 캡슐화.
- **모바일 하단 sticky CTA**(스크롤 시 `정기구독 시작하기`/`첫 구매` 고정, 키 섹션 중복 숨김)
- **SEO 메타** 쇼핑몰/구독 중심 개선(title/description/OG) + 상품 JSON-LD(Product/Offer)
- 접근성(라벨·대비·키보드), 성능(이미지 최적화·CLS 방지)

### 4-8. 디자인
프리미엄 목장 톤 유지 + "데이터 기반 추천·구독 전환" 느낌 일부. **과한 사이버/네온 금지**(우유 신뢰감 우선).
